# RoboTrader 블로그 시리즈 구성안

## 시리즈 개요
Python 기반 자동매매 시스템 RoboTrader의 설계와 구현 과정을 6편의 시리즈로 소개합니다.

## 목표 독자
- 주식 자동매매에 관심 있는 개발자
- Python으로 트레이딩 시스템을 구축하려는 분
- 실전 매매 전략 구현에 관심 있는 트레이더

---

## 시리즈 1: 눌림목 전략의 비밀 📊
**부제**: 거래량 급감이 알려주는 반등 신호

### 주요 내용
- 눌림목 캔들패턴이란?
- 4단계 패턴 분석 (상승 → 조정 → 지지 → 돌파)
- 이등분선의 역할과 계산법
- 거래량 분석 기법 (기준 거래량 vs 저거래량)
- 신호 강도별 목표 수익률 설정
- 실제 차트 사례 분석

### 핵심 메시지
"적은 돈으로 올랐다면, 적은 돈으로 내려온다"

### 예상 분량
2,500자

## 시리즈 2: 자동매매 시스템 설계 💻
**부제**: 비동기 처리로 실시간 매매를 정복하다

### 주요 내용
- 전체 시스템 아키텍처
- 비동기(asyncio) 프로그래밍의 힘
- 6개 태스크 병렬 처리 전략
- Manager 패턴으로 도메인 분리
- 상태 관리 시스템 (FSM)
- 긴급 포지션 동기화

### 핵심 코드
```python
async def main():
    tasks = [
        _data_collection_task(),      # 10초 주기
        _order_monitoring_task(),     # 10초 주기
        _trading_decision_task(),     # 5초 주기
        _system_monitoring_task(),    # 30초 주기
        _telegram_task()              # 실시간 알림
    ]
    await asyncio.gather(*tasks)
```

## 시리즈 3: 데이터 수집 전략 📡
**부제**: API 30건 제한을 우회하는 기술

### 주요 내용
- 한국투자증권 API 제약사항
- 완성된 봉 vs 진행중인 봉
- historical_data + realtime_data 결합 전략
- 1분봉 연속성 검증
- 3분봉 TimeFrame 변환
- 데이터 품질 모니터링

### 핵심 기술
- `get_full_trading_day_data_async()`: 09:00부터 전체 수집
- 선정 시점 이전 데이터만 필터링 (미래 데이터 차단)
- 당일 데이터만 추출 (전날 데이터 혼입 방지)

## 시리즈 4: 대량 종목 관리의 함정과 해법 ⚠️
**부제**: 70개 종목, 초당 20개 API 제한 극복기

### 주요 내용
- 70개 종목 선정 시 발생하는 문제
- API 호출 한도 분석 (초당 20개)
- 고정 배치 vs 동적 배치
- DynamicBatchCalculator 구현
- 실시간 성공률 모니터링

### Before & After
| 방식 | 배치 크기 | 소요 시간 | API 속도 | 결과 |
|------|----------|----------|---------|------|
| 고정 | 20개 | 0.8초 | 175개/초 | ❌ 875% 초과 |
| 동적 | 10개 | 7.0초 | 20개/초 | ✅ 정상 |

## 시리즈 5: 리스크 관리의 정석 🛡️
**부제**: 손절은 기술이다, 생존의 기술

### 주요 내용
- 다층 손절 체계
  - 이등분선 이탈 (-0.2%)
  - 진입 양봉 저가 이탈
  - 지지 저점 이탈
  - 손익비 기반 손절
- 15시 정각 시장가 일괄 청산
- 쿨다운 시스템 (25분 재매수 금지)
- 중복 신호 차단 (3분봉 정규화)
- 자금 관리 (종목당 10%)

### 핵심 통계
- 매수 제한 조건 8가지
- 손절 조건 4가지
- 익절 조건 2가지

### 예상 분량
2,800자

## 시리즈 6: 실전 운영 노하우 🚀
**부제**: 장 시작부터 차트 생성까지 완전 자동화

### 주요 내용
- 장 시작 초기화 프로세스
- 텔레그램 실시간 모니터링
- 로깅 및 디버깅 전략
- 장마감 후 차트 자동 생성
- 포지션 동기화 (계좌 ↔ 시스템)
- 에러 복구 전략

### 운영 타임라인
```
09:00  시스템 시작 → API 인증
09:00~12:00  매수 판단 활성화
12:00~15:00  보유 종목 매도 판단만
15:00  시장가 일괄 청산
15:30  장 마감
16:00  차트 자동 생성 및 텔레그램 전송
```
